Premiers pas

Installez le compilateur AFRILANG, exécutez votre premier programme, créez un projet et découvrez les commandes essentielles de la CLI.

Prérequis

  • Linux (testé sur Ubuntu/Debian) — macOS/Windows via toolchain compatible
  • CMake ≥ 3.16, g++ avec support C++17 (C++20 pour async)
  • Git pour cloner le dépôt

Installation du compilateur

Terminal
git clone https://github.com/MaximeKELI/AFRILANG.git
cd AFRILANG
mkdir build && cd build
cmake ..
cmake --build . -j$(nproc)

# Vérifier
./afrilang --version
./afrilang run ../examples/hello.afr

Le binaire afrilang se trouve dans build/. Ajoutez-le à votre PATH ou créez un lien symbolique dans /usr/local/bin.

Premier programme

Créez hello.afr :

say "Bonjour, AFRILANG!"

repeat 3 times
    say "Hello"
end
Exécution
afrilang run hello.afr

La commande run transpile, compile avec g++ et exécute le binaire en une seule étape.

Créer un projet

afrilang init mon_app
cd mon_app
afrilang build
afrilang run src/main.afr

init génère afrilang.toml (manifeste) et src/main.afr. build compile le projet ; test lance les blocs test intégrés.

Manifeste afrilang.toml

[project]
name = "mon_app"
version = "0.1.0"

[dependencies]
math = "0.1.0"

Installez les dépendances avec afrilang pkg install. Les paquets certifiés sont listés sur la page Paquets.

Commandes CLI essentielles

CommandeDescription
afrilang run fichier.afrCompiler et exécuter
afrilang buildCompiler le projet (afrilang.toml)
afrilang check fichier.afrVérifier sans générer de binaire
afrilang testExécuter la suite de tests du projet
afrilang fmt -w fichier.afrFormater le source
afrilang lint fichier.afrAnalyse statique
afrilang replREPL interactif
afrilang lspServeur Language Server (stdio)
afrilang pkg add mathAjouter un paquet certifié
afrilang build --target wasm32Cross-compilation WebAssembly
afrilang explain fichier.afrMode pédagogique ligne par ligne

Extension IDE

Installez l'extension AFRILANG (v1.4+) depuis VS Code, Cursor ou Open VSX. Elle fournit coloration, diagnostics LSP, complétion FR/EN, symboles, code actions et formatage. Configurez le chemin du compilateur si nécessaire : afrilang.compilerPath.

Essayer sans installation

Utilisez le playground web pour exécuter du code AFRILANG directement dans le navigateur (compilation côté serveur).
