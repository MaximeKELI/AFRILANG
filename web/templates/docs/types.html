Système de types

AFRILANG est typé statiquement : chaque expression a un type vérifié à la compilation. Les types transpilent vers des équivalents C++17 standard.

Types primitifs

Optionnels et null-safety

create name text? = nothing

if name is defined then
    say name
else
    say "No name"
end

function findUser(id number) returns text?
    if id is less than 0 then
        return nothing
    end
    return "User" + id
end

Type Result

function parseAge(s text) returns number or error
    create n = toNumber(s)
    if n is less than 0 then
        return error "invalid age"
    end
    return n
end

create r = parseAge("25")
if r is error then
    say r.message
else
    say r.value
end

Inférence et annotations

L'inférence fonctionne sur les littéraux, les listes et les appels génériques. Les paramètres de fonction et les champs de classe doivent être typés explicitement (sauf avec génériques). 

create nums = list of 1, 2, 3        # list of number
create m = map of "x" to 1             # map text to number

function identity(x T) returns T
    return x
end

say identity(42)        # inféré number
say identity(42) # explicite

Enums avec payload

enum HttpResult
    case Ok with body text
    case NotFound
    case Error with code number, message text
end

create r = HttpResult.Error with 500, "Server error"
