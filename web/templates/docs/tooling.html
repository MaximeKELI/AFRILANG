Outils et écosystème

CLI, LSP, REPL, formatter, linter, gestionnaire de paquets, debug adapter, extension IDE, playground web et CI — la toolchain complète d'AFRILANG.

CLI — référence rapide

CommandeRôle
afrilang build [--target …]Compile le projet
afrilang run fichier.afrCompile + exécute
afrilang checkAnalyse sans codegen
afrilang testSuite de tests
afrilang fmt [-w]Formateur de code
afrilang lintLinter statique
afrilang replREPL v2 (:load, :type, :history)
afrilang lspLanguage Server (stdio JSON-RPC)
afrilang docGénération de documentation
afrilang explainMode pédagogique
afrilang benchmarkBenchmarks de compilation
afrilang servePlayground statique (port 8080)
afrilang debugDebug adapter (DAP)

Language Server (LSP 1.1)

Le serveur LSP fournit :

  • Diagnostics en temps réel avec codes d'erreur stables (E3002, …)
  • Hover enrichi (types, documentation)
  • Go to definition, find references
  • Complétion bilingue FR/EN
  • Symboles document et workspace
  • Code actions (quick fixes)
  • Formatage via textDocument/formatting
# Lancer le LSP (utilisé par l'extension IDE)
afrilang lsp

Extension VS Code / Cursor

Version 1.4.0 — disponible sur Open VSX et en installation locale depuis vscode-afrilang/.

  • Coloration syntaxique AFRILANG
  • Intégration LSP complète
  • Icônes et thème de fichiers .afr
  • Snippets et commandes (Run, Format)

Gestionnaire de paquets

afrilang pkg list --blessed     # paquets certifiés
afrilang pkg search math
afrilang pkg add math
afrilang pkg install            # depuis afrilang.toml
afrilang pkg publish            # publier localement

Résolution semver 2.0, lockfile afrilang.lock, vérification SHA256 pour les paquets certifiés. Registre officiel : page Paquets.

Debug

Le codegen insère des directives #line pour mapper le C++ généré vers les sources .afr. Le debug adapter (DAP) communique avec GDB/MI et affiche les variables typées avec métadonnées Afrilang.

Cross-compilation

afrilang build --target native        # défaut
afrilang build --target linux-x64
afrilang build --target linux-arm64
afrilang build --target wasm32        # Emscripten

Playground web

Deux options : le serveur statique C++ (afrilang serve) ou ce site Django avec le playground interactif (API REST : POST /api/run/, POST /api/fmt/).
